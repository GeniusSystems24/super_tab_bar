01 · Desktop

The full tab strip

Click a tab to switch (pages preserve their state). Hover ~0.5 s for a mini-page preview. Right-click for the context menu. Drag an unpinned tab to reorder. Use ＋ to add and ▾ for the overflow list. Pinned tabs are icon-only and anchored to the start edge.

02 · Behaviors

Per-tab behavior types

Each tab declares a SuperTabBehavior that governs which UI actions it exposes — right-click any tab above to see the menu adapt.

requiredPinned

Always pinned. The UI hides close, unpin and duplicate — but controller.close(id) still works programmatically.

Like normal, but never duplicated. Re-adding with the same uniqueKey just re-selects the existing tab.

03 · Mobile · New in 2.1

Compact mode & the tab switcher

On phones, set compact:true to hide the strip. A FloatingActionButton opens a full-screen grid of tab thumbnails. Tap a thumbnail to switch, long-press-drag one onto another to reorder, and × to close (dirty tabs confirm first).

Dirty-aware back navigation

With closeTabOnBack:true a back gesture closes the current tab — but only when it isn't dirty. Try the ← button on the phone, and toggle the "Clean/Dirty" chip.

04 · Responsive · New in 2.2 & 2.3

Auto-compact breakpoint & built-in FAB

Drag the handle or slider to resize the frame. With allowAutoCompact:true the strip collapses on its own once the width reaches compactWidth (600 px). useCompactFloatingActionButton:true then shows a built-in FAB that opens the switcher — no extra scaffolding.
